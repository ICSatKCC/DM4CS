Consider the LinearSearch algorithms Algorithm 9.1.9 and Algorithm 9.1.10 that find a target value in an array of \(n\) integers. The algorithm looks at each integer in turn, comparing it to the target. Here, the size of the problem is the number of integers stored in in the list. The basic operation is to compare an integer’s value to that of the target. As with MaximumElement above, it is reasonable to assume that it takes a fixed amount of time to do one such comparison, regardless of the value of the two integers or their positions in the array.
